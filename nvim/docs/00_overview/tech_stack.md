# Tech Stack

## Runtime

- Neovim Lua → primary configuration runtime
- Lazy.nvim → plugin manager bootstrap and plugin spec loader
- Mason.nvim → external tool and LSP package management

## Frontend

- none → editor configuration only

## Backend

- none → no application server

## Storage

- `lazy-lock.json` → plugin version lockfile
- `spell/*` → local spellfile data

## Infra

- git → dotfiles versioning
- shell scripts/Lua scripts → local validation helpers
